`run_turn(...)` and `continue_turn(...)` currently drop `BootstrapArtifacts.history_warnings` on entry to `execute_turn` (`crates/pi_lynx_sdk/src/runtime.rs:109-116`). That means a host using the documented high-level runtime API cannot observe recoverable transcript diagnostics such as dropped custom blocks or other warning-producing history conversions unless it bypasses the main API and calls `bootstrap_turn(...)` manually. This leaves the primary embed surface unable to satisfy the transcript warning contract.

Suggested fix: thread `history_warnings` through `TurnResult`/`TurnResultMetadata` or emit them as embed events from `execute_turn`, and add a runtime test that warning-producing transcripts remain observable through `run_turn(...)`/`continue_turn(...)`.

## Resolution
Preserved BootstrapArtifacts.history_warnings through runtime execution by adding TurnResult.history_warnings and covering the shared run/continue artifact execution paths with runtime tests.
